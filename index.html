Asynchronous Programming and Callbacks

JavaScript is single-threaded, meaning it executes one operation at a time. However, asynchronous programming enables tasks like I/O operations, network requests, and timers to run in the background without blocking the main thread.

What are Callbacks?

  • A callback is a function passed as an argument to another function and executed later.
  • Commonly used in asynchronous operations to ensure certain code runs only after another task completes.

Example: Using Callbacks

function greet(name, callback) {
  console.log('Hello ' + name); // Print greeting message
  callback(); // Call the callback function
}

function logEnd() {
  console.log('Function execution ended.');
}

// Call greet with a callback function
greet('Alice', logEnd);

// Output:
// Hello Alice
// Function execution ended.

Example: Simulating Asynchronous Operations

function processUserData(userId, callback) {
  setTimeout(() => {
    if (userId) {
      callback(null, { id: userId, name: "John Doe" }); // Success
    } else {
      callback("User ID not provided", null); // Error
    }
  }, 1500); // Simulate delay
}

// Example usage
processUserData(1, (error, user) => {
  if (error) {
    console.log("Error:", error); // Log error message
  } else {
    console.log("User Data:", user); // Log user data
  }
});

Promises and Async/Await in JavaScript

Promises

A Promise in JavaScript is an object representing the eventual completion (or failure) of an asynchronous operation and its resulting value. It offers a cleaner and more robust way to handle asynchronous operations compared to traditional callbacks, which can lead to callback hell.

Key Concepts

  • Pending: The initial state; neither fulfilled nor rejected.
  • Fulfilled: The operation completed successfully.
  • Rejected: The operation failed.

Creating a Promise

// Creating a new Promise object
const myPromise = new Promise((resolve, reject) => {
  const success = true; // Change this to `false` to simulate failure

  if (success) {
    resolve("Operation succeeded!"); // If success is true, resolve the promise
  } else {
    reject("Operation failed."); // If success is false, reject the promise
  }
});

Handling Promises

You can handle promises using .then() and .catch() methods:

then() Method

The .then() method is used to handle the resolved value of a Promise. It can take up to two arguments:

  1. A callback function for the fulfilled case (when the Promise is resolved).
  2. An optional callback function for the rejected case (when the Promise is rejected).

catch() Method

The .catch() method handles the rejected value of a Promise and takes one argument: a callback function for the rejected case.

Example

myPromise
  .then((message) => { 
    console.log("Success:", message); // Runs if the promise is resolved
  })
  .catch((error) => { 
    console.log("Error:", error); // Runs if the promise is rejected
  });

Example: Fetch Data with Promises

Code

const fetchData = (url, callback) => {
  const myPromise = new Promise((resolve, reject) => {
    fetch(url) // Fetch data from the given URL
      .then((response) => {
        if (response.ok) { // Check if the response is successful (status 200-299)
          return response.json(); // Convert response to JSON
        } else {
          reject("Failed to fetch data: " + response.statusText); // Reject if response is not OK
        }
      })
      .then((data) => {
        resolve(data); // Resolve the promise with fetched data
      })
      .catch((error) => {
        reject("Network error: " + error.message); // Reject for network errors
      });
  });

  myPromise
    .then((data) => {
      callback(null, data); // No error, pass data to the callback
    })
    .catch((error) => {
      callback(error, null); // Pass the error to the callback function
    });
};

// Example usage
const url = "https://jsonplaceholder.typicode.com/posts";
fetchData(url, (error, data) => {
  if (error) {
    console.error("Error:", error); // Logs error if fetching fails
  } else {
    console.log("Fetched data:", data); // Logs the fetched data if successful
  }
});

Why Use Promises?

1Ô∏è‚É£ Avoiding Callback Hell

  • Promises make the code easier to read by avoiding deeply nested callbacks.
  • You can chain .then() and .catch() for sequential operations.

2Ô∏è‚É£ Simplified Error Handling

  • Centralized error handling using .catch().
  • Avoids the need for duplicating error-checking logic inside nested callbacks.

3Ô∏è‚É£ Better Control Over Asynchronous Operations

  • Promises allow developers to resolve() or reject() based on conditions, offering structured control.

4Ô∏è‚É£ Avoiding "Callback Inversion"

  • Promises provide a more predictable and linear execution flow compared to callbacks.

5Ô∏è‚É£ Enhanced Debugging

  • Promises work seamlessly with modern debugging tools, offering improved stack traces.

Async/Await: A Cleaner Alternative

Introduction

async/await syntax provides an even cleaner way to work with Promises. It makes asynchronous code look synchronous, improving readability.

Key Features

Async Keyword

  • Declares an asynchronous function that implicitly returns a Promise.
  • Allows usage of the await keyword within the function.

Await Keyword

  • Pauses execution until the Promise is resolved or rejected.
  • Can only be used inside an async function.

Example: Fetch Data Using Async/Await

const fetchDataAsync = async (url) => {
  try {
    const response = await fetch(url);
    if (!response.ok) {
      throw new Error("Failed to fetch data: " + response.statusText);
    }
    const data = await response.json();
    console.log("Fetched data:", data);
  } catch (error) {
    console.error("Error:", error.message);
  }
};

// Example usage
fetchDataAsync("https://jsonplaceholder.typicode.com/posts");

    console.log(result2);
  } catch (error) {

Closures, Event Loop, and Undici in Node.js

Closures

A closure is a function that remembers the environment in which it was created. It allows a function to access variables from its outer (enclosing) scope even after the outer function has finished executing.

Example

function outerFunction() {
  const outerVariable = 'I am from the outer scope';

  function innerFunction() {
    console.log(outerVariable); // Accesses outerVariable even after outerFunction is done
  }

  return innerFunction;
}

const myClosure = outerFunction();
myClosure(); // Logs: 'I am from the outer scope'

Explanation

  • outerFunction creates a local variable outerVariable and defines innerFunction.
  • innerFunction has access to outerVariable even after outerFunction has returned.
  • myClosure holds the innerFunction, which still has access to outerVariable due to the closure.

The Event Loop

The Event Loop handles JavaScript‚Äôs asynchronous operations, ensuring non-blocking code execution. It processes tasks in the Call Stack, Web APIs, Callback Queue, and Microtask Queue (Promises) in a cyclic manner.

Example

console.log("Start");

setTimeout(() => console.log("Timeout callback"), 0);

Promise.resolve().then(() => console.log("Promise resolved"));

console.log("End");

Output

Start
End
Promise resolved
Timeout callback

Explanation

  • Promises are executed before setTimeout because they are in the Microtask Queue, which has a higher priority than the Callback Queue.

2Ô∏è‚É£ Events Module in Node.js

The events module is a core part of Node.js that enables handling and working with events in an asynchronous, event-driven manner. It is at the heart of Node.js‚Äôs non-blocking architecture.

What is an Event in Node.js?

An event is essentially a signal that something has happened in the application. For example:

  • A user clicks a button (in a UI application).
  • A file finishes downloading.
  • A server receives a request.

In Node.js, events help implement its non-blocking, asynchronous runtime. When an event occurs, associated listeners (or callbacks) respond to them.

The Event Module

The events module provides the EventEmitter class, which allows you to:

  1. Emit an Event: Trigger an event to signal that something happened.
  2. Listen to an Event: Register a listener that executes when the event is triggered.

How Events and Listening Work

Example Code

  console.log(`Hello, ${name}!`);
});

// Emit the 'greet' event

What‚Äôs Happening Here?

  1. Event Emitter Creation:
    An instance (myEmitter) is created from the EventEmitter class.

  2. Registering a Listener:
    The .on() method listens for a specific event ('greet' in this case). When the event is triggered, the provided callback function runs.

  3. Emitting the Event:
    The .emit() method triggers the event, optionally passing along arguments ('MD' here, passed to the listener‚Äôs function).

CRUD Operations with Express.js

CRUD (Create, Read, Update, Delete) operations are the backbone of web applications, allowing interaction with data. This guide showcases how to perform CRUD operations in a simple Express.js application.

Code Example: CRUD Operations

// Import the Joi module for data validation.

// Import the 'express' module.

// Use the express.json() middleware to parse JSON bodies.

  { id: 1, name: 'course1' },
  { id: 2, name: 'course2' },
  { id: 3, name: 'course3' }
];

// GET - Fetch all courses
app.get('/api/courses', (req, res) => {
  res.send(courses);
});

// POST - Add a new course
app.post('/api/courses', (req, res) => {
  const { error } = validateCourse(req.body);
  if (error) return res.status(400).send(error.details[0].message);

  const course = { id: courses.length + 1, name: req.body.name };
  courses.push(course);
  res.send(course);
});

// PUT - Update a course by ID
app.put('/api/courses/:id', (req, res) => {
  const course = courses.find(c => c.id === parseInt(req.params.id));
  if (!course) return res.status(404).send('Course not found.');

  const { error } = validateCourse(req.body);
  if (error) return res.status(400).send(error.details[0].message);

  course.name = req.body.name;
  res.send(course);
});

// DELETE - Delete a course by ID
app.delete('/api/courses/:id', (req, res) => {
  const course = courses.find(c => c.id === parseInt(req.params.id));
  if (!course) return res.status(404).send('Course not found.');

  const index = courses.indexOf(course);
  courses.splice(index, 1);
  res.send(course);
});

// Validation function using Joi
function validateCourse(course) {
  const schema = Joi.object({ name: Joi.string().min(3).required() });
  return schema.validate(course);
}

// Start the server
const port = process.env.PORT || 3000;
app.listen(port, () => console.log(`Listening on port ${port}...`));
